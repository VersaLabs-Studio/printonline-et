  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  recipient_id TEXT NOT NULL,
  order_id UUID REFERENCES orders(id) ON DELETE CASCADE,
  message TEXT NOT NULL,
  is_admin BOOLEAN DEFAULT false,
  read_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),

CREATE INDEX idx_messages_order ON messages(order_id);
CREATE INDEX idx_messages_sender ON messages(sender_id);

## User Clarifications (Confirmed 2026-04-16)

### ✅ Delivery Approach
**Decision**: Flat-rate per sub-city based on distance from PrintOnline HQ
- Base rate calculated from HQ location
- Further sub-cities = higher fee
- Pickup option = 0 ETB

### ✅ Account Deletion
**Decision**: Immediate soft-delete
- Set `is_active: false` on customer_profiles
- Send deletion confirmation email
- No confirmation window

### ⚠️ Chapa Receipt Issue
**Problem**: Receipt displays briefly during hosted checkout, then disappears completely
**Analysis**: Likely Chapa hosted page issue, but we can implement workarounds:
1. **Capture receipt data BEFORE redirect** - Store in order metadata
2. **Webhook payload** - Chapa sends receipt data to webhook, store it
3. **Order confirmation page** - Fetch from our database, not Chapa
4. **Email receipt** - Send receipt via email as backup

**Solution Approach**:
- Enhance webhook handler to store full receipt data
- Display receipt from our database on order confirmation
- Add "Download Receipt" button that generates PDF from stored data

### ⚠️ Mac Chrome Performance Issues
**Problem**: Full crash, stuttering, disappearing UI, unresponsiveness, buggy behavior
**Severity**: CRITICAL - This is a showstopper for Mac users

**Suspected Causes**:
1. **Framer Motion animations** - GPU acceleration issues on Mac Chrome
2. **Memory leaks** - React components not cleaning up properly
3. **Large bundle size** - Too much JavaScript parsing
4. **Image optimization** - Unoptimized images causing memory spikes
5. **React Query over-fetching** - Excessive re-renders

**Investigation Plan**:
1. Run Lighthouse audit on Mac Chrome
2. Chrome DevTools Memory Profiler
3. Chrome DevTools Performance tab
4. Bundle size analysis (`pnpm build --analyze`)
5. Check for animation-heavy components

**Quick Fixes to Try**:
- Disable Framer Motion on low-end devices (`prefers-reduced-motion`)
- Lazy load heavy components
- Optimize images with `next/image`
- Reduce concurrent animations
- Add `will-change` CSS hints

# Plan: Disable Hire-a-Designer & Mandate Design Uploads

## Goal
1. Fully remove the "hire a designer" / "I don't have a Design" option from the product detail page.
2. Enforce that **every** product added to cart has at least one design file uploaded.
3. Block checkout on the cart and order-summary pages if any item is missing design files.

## Files to Change

### 1. `components/product/ProductOrderForm.tsx`
- **Remove UI:**
  - Delete the "I don't have a Design" toggle button (lines 505–543).
  - Delete the design-package tier selector block (lines 547–609).
- **Remove state & imports:**
  - Remove `hireDesigner`, `selectedPackageId`, `setHireDesigner`, `setSelectedPackageId` state.
  - Remove unused imports: `User`, `CheckCircle2`, `Crown`, `Sparkles`, `Star`, `DESIGN_PACKAGES`, `getDesignPackageById`, `formatDesignPackageLabel`, `DesignPackageId`.
- **Update price calculation:**
  - Remove `designPackagePrice` from `totalPrice`.
- **Update validation (`handleProceed`):**
  - Change the design check from `designFiles.length === 0 && !hireDesigner` to simply `designFiles.length === 0`.
  - Remove the `hireDesigner && !selectedPackageId` validation block.
  - Update error message to: "Please upload your design file(s)."
- **Update cart payload:**
  - Remove `designPackageId`, `designPackageName`, `designPackagePrice`, and `hireDesigner` from the `cartItem` object.
  - Remove the logic that adds "Design Package" to `humanReadableOptions`.

### 2. `context/CartContext.tsx`
- **Backward-compat cleanup on load:**

  - Remove `item.designPackagePrice || item.designerFee || 0` from the reduce accumulator (keep only `unitPrice * quantity + priorityPrice`).
- **Type updates (optional but recommended):**

### 3. `app/cart/page.tsx`

  - If valid, navigate to `/order-summary`.
- **Wire guard to checkout buttons:**
  - Replace the mobile sticky `<Link href="/checkout">` with a `<Button onClick={handleCheckout}>`.
  - Pass the same guard callback down to `<CartSummary>` so the desktop "Proceed to Checkout" button is also gated.

## Backward Compatibility
- Existing carts in `localStorage` that used the designer feature will have those fields stripped on next load. Because they won't have actual `File` objects in `localFiles`, the checkout guard will block them and prompt the user to re-add the item with an uploaded design.

## Out of Scope
- CMS admin pages (`app/(cms)/cms/products/[id]/designer-fees`, API routes, DB tables) are **not** being deleted; only the customer-facing purchase flow is being disabled.

## Validation Steps
1. Open any product detail page → confirm "I don't have a Design" button and package tiers are gone.
2. Try clicking "Add to Cart" without uploading files → should get a toast error.
3. Upload a file and add to cart → should succeed and redirect to `/cart`.
4. On `/cart`, click "Proceed to Checkout" → should navigate to `/order-summary`.
5. Remove files from memory (e.g., clear component state or refresh page) → checkout button should show error toast.
6. On `/order-summary`, confirm order payload no longer contains designer fields.

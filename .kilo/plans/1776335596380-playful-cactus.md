# Print Online Ethiopia v3.5 Testing Plan

## Overview

This document outlines the verification and testing process for all v3.5 features. All implementation work is complete; this plan focuses on verification.

## Features to Test

### 1. Account Deletion
**Implementation Location**: `app/api/account/delete/route.ts`, `lib/auth-client.ts`

**Test Cases**:
- [ ] Soft-delete flow: User requests deletion, account marked inactive
- [ ] Confirmation email sent to user's email
- [ ] Admin hard delete from CMS works
- [ ] Deleted user cannot log in
- [ ] Deleted user's order history preserved

### 2. Designer Fee
**Implementation Location**: `lib/supabase/client.ts`, `components/product/`

**Test Cases**:
- [ ] Product creation stores `hireDesignerFee` correctly
- [ ] Checkout applies designer fee to line items
- [ ] Database storage uses `selected_options.Service = "Pana Designer"` and `selected_options.hireDesignerFee`
- [ ] Fee displays correctly in order summary

### 3. File Upload
**Implementation Location**: `components/order/OrderItemDesignAssets.tsx`

**Test Cases**:
- [ ] File upload succeeds for design assets
- [ ] Uploaded files display in order details
- [ ] File download functionality works
- [ ] Files are properly associated with order items

### 4. Order Status
**Implementation Location**: `components/order/OrderStatusTracker.tsx`, `components/order/ConfirmationDetails.tsx`

**Test Cases**:
- [ ] All status transitions work in CMS
- [ ] UI displays status correctly
- [ ] Status history is tracked
- [ ] Email notifications fire on status change

### 5. Message Portal
**Implementation Location**: `components/chat/MessagePortal.tsx`, `components/chat/MessageThread.tsx`

**Test Cases**:
- [ ] Real-time messaging works (Supabase subscription)
- [ ] Threading by `order_item_id` functions correctly
- [ ] Media exchange (images/files) works
- [ ] Admin can view all messages
- [ ] Message delivery latency < 2 seconds

### 6. Pending Orders
**Implementation Location**: `app/checkout/page.tsx`, `lib/email.ts`

**Test Cases**:
- [ ] Payment failure creates `payment_pending` order
- [ ] Email retry sends after failure
- [ ] Auto-cancellation after 24 hours works
- [ ] Idempotency keys prevent duplicate charges

### 7. Chapa Webhook Handler
**Implementation Location**: `lib/chapa.ts`

**Test Cases**:
- [ ] Webhook captures receipt data before redirect
- [ ] Idempotent webhook processing
- [ ] Receipt data stored correctly

### 8. Delivery Zone Fallback
**Implementation Location**: `lib/supabase/client.ts`, `lib/validations/checkout.ts`

**Test Cases**:
- [ ] Unknown sub-city defaults to 300 birr
- [ ] Warning toast displays for unknown zones
- [ ] `DELIVERY_ZONES` lookup table works
- [ ] Quantity matrix validation functions

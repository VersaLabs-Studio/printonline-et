# PrintOnline Ethiopia v3.5 Implementation Plan

## Overview
This document outlines the comprehensive implementation plan for v3.5 of printonline-et, addressing all 10 requirements from the specification.

## Architecture Overview

### Key Directories
- `app/` - Next.js App Router pages and layouts
- `components/` - Reusable UI components
- `context/` - React contexts (CartContext, ThemeProvider, etc.)
- `hooks/` - Custom hooks (data fetching, domain logic, UI)
- `lib/` - Core libraries, auth, chapa integration, Supabase client
- `types/` - TypeScript type definitions
- `supabase/` - Database schema, storage, admin functions
- `content/` - Static content

### Data Models
- **Product**: Core product definition (lib/products.ts) - used throughout app
- **Product (types/product.ts)**: Extended schema with Order, Address, CartItem types
- **Order**: Database order schema with status, pricing, customer info
- **OrderItem**: Line items with design assets, options, pricing
- **CustomerProfile**: User profile with address, sub-city, city info

## Requirement 2: Price and Quantity Matrix for Delivery Costs on Larger Orders

### Implementation
**Files to Modify:**
- `lib/supabase/client.ts` - Add `calculateDeliveryWithMatrix` function
- `lib/validations/checkout.ts` - Add quantity-based delivery rules
- `app/order-summary/page.tsx` - Apply matrix pricing

**Approach:**
1. Create `product_pricing_matrix` table for delivery cost tiers
2. Define matrix: quantity brackets → delivery cost multipliers
3. Example: 
   - 1-5 items: base delivery
   - 6-10 items: 80% of base
   - 11-20 items: 60% of base
   - 20+ items: free delivery

**Database Schema:**
```sql
product_pricing_matrix (
  id: uuid PRIMARY KEY,
  product_id: uuid REFERENCES products(id),
  matrix_key: text, -- e.g., "delivery_tier"
  matrix_label: text, -- e.g., "Quantity 1-5"
  price: number, -- delivery multiplier
  is_active: boolean
)
```

```sql

  sender_id: text, -- user or admin
```

## Requirement 10: Saved Order Pending Status for Failed Payments with Email Notification

### Implementation
**Files to Modify:**
- `lib/supabase/client.ts` - Add `savePendingOrder` function
- `app/checkout/page.tsx` - Save pending order on payment failure
- `lib/email.ts` - Add email notification function
- `lib/validations/cms.ts` - Email template schema

**Approach:**
1. On payment initiation, save order as "pending"
2. On payment failure, keep order in pending state
3. Send email notification to user
4. Allow user to retry payment from account

**Order Status Flow:**
- New status: `payment_pending`
- On success: proceed to `order_confirmed`
- On failure: remain `payment_pending` with email notification

**Database Schema:**
```sql
orders (
  ...
  status: string, -- includes 'payment_pending'
  payment_attempts: integer,
  last_payment_error: text,
  email_notified_at: timestamp
)
```

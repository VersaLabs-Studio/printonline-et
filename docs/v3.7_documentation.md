## Executive Summary

Version 3.7 makes the **storefront fully dynamic** and the **auth system production-ready**. Every previously hardcoded homepage section (hero slideshow, deals, testimonials) is now CMS-configurable. The auth system adds Google, Facebook, and TikTok OAuth, real email verification, and proper password recovery. Cookie/privacy compliance is implemented. Product metadata gains manual quantity entry and interval configuration. The CMS save UX is refined to "save and stay."

### What Changed Since v3.6

| Feature | Before v3.7 | After v3.7 |
|---------|-------------|------------|
| Hero Slideshow | 3 hardcoded slides in component | CMS-configurable, max 4, API-backed |
| Special Deals | 3 hardcoded deals | CMS-configurable, max 3, API-backed |
| Testimonials | 3 hardcoded entries | CMS-configurable, API-backed |
| CMS Save Behavior | Redirects to list on save | Stays on edit page (Save & Stay) |
| Cookie/Privacy | No banner, broken links | Cookie consent banner + policy pages |
| Product Images | Decorative drag handle only | Functional drag-to-reorder + primary |
| Quantity Entry | Hardcoded preset dropdown | Per-product configurable (manual/interval/presets) |
| Auth | Email/password only | Google + Facebook + TikTok OAuth |
| Email Verification | Simulated (commented out) | Real email verification flow |
| Password Reset | Simulated (setTimeout) | Real SMTP password reset |
| Newsletter | Email signup form | Contact us form |
| Footer | 4 phone numbers, dead links | 3 phone numbers, working links |
| New DB Tables | 5 v3.6 tables | +5 new tables (hero_slides, homepage_deals, homepage_testimonials, privacy_policies, product quantity columns) |
| New API Routes | 16 routes | +13 routes (total 29 CMS/public routes) |
| New Pages | 65 | 82 (17 new pages incl. CMS + policy + auth) |

## What's New in Version 3.7

### F1: Dynamic Hero Slideshow

**Category:** Homepage CMS Enhancement  
**Impact Level:** High — Admin team can now manage hero slides through CMS  
**User-Facing Change:** Hero slideshow can be updated without code changes

#### What Changed

The hero slideshow previously had 3 hardcoded slides in `HeroSection.tsx`. Now slides are stored in the `hero_slides` database table and managed through the CMS. The component falls back to hardcoded data if the API is unavailable.

#### Features

- **CMS Management** — Add, edit, delete, and reorder hero slides at `/cms/settings/hero`
- **Max 4 Slides** — Enforced at CMS level with visible counter
- **Image Upload** — URL-based image management
- **CTA Customization** — Per-slide button text and link target
- **Active Toggle** — Enable/disable slides without deleting
- **Graceful Fallback** — Hardcoded defaults preserved if API fails

#### Where You'll See This

- **CMS → Settings → Hero Slides** — Manage slideshow content
- **Homepage** — Slideshow renders active slides from CMS

## Migration Guide

### Prerequisites

- Access to Supabase Dashboard (SQL Editor)
- Service role or postgres role permissions
- All prior migrations (001-024) already applied

### Migration Files

Run these SQL scripts in the Supabase SQL Editor **in this exact order**:

| # | Migration File | What It Does | Safe to Re-run? |
|---|----------------|--------------|-----------------|
| 1 | `025_homepage_hero_slides.sql` | Creates `hero_slides` table + indexes + RLS + trigger + seed | Yes (uses IF NOT EXISTS) |
| 2 | `026_homepage_deals.sql` | Creates `homepage_deals` table + indexes + RLS + trigger + seed | Yes (uses IF NOT EXISTS) |
| 3 | `027_homepage_testimonials.sql` | Creates `homepage_testimonials` table + indexes + RLS + trigger + seed | Yes (uses IF NOT EXISTS) |
| 4 | `028_product_quantity_config.sql` | Adds columns to `products` table | Yes (uses IF NOT EXISTS) |
| 5 | `029_cookie_consent.sql` | Creates `privacy_policies` table + indexes + RLS + trigger + seed | Yes (uses IF NOT EXISTS) |

### Seed Data

All seed data is included in the migration files (hero_slides: 3 slides, homepage_deals: 3 deals, homepage_testimonials: 3 testimonials, privacy_policies: 3 policies — privacy, terms, cookie).

No separate seed files needed.

### Verification Queries

```sql
-- 1. Verify hero_slides (expect 3 rows)
SELECT title, image_url, display_order, is_active FROM hero_slides ORDER BY display_order;

-- 2. Verify homepage_deals (expect 3 rows)
SELECT title, badge_text, badge_color, display_order FROM homepage_deals ORDER BY display_order;

-- 3. Verify homepage_testimonials (expect 3 rows)
SELECT name, rating, display_order FROM homepage_testimonials ORDER BY display_order;

-- 4. Verify products has new columns (expect 3 rows)
SELECT column_name, data_type, column_default
FROM information_schema.columns
WHERE table_name = 'products'
AND column_name IN ('manual_quantity_entry', 'quantity_interval', 'quantity_presets');

-- 5. Verify privacy_policies (expect 3 rows)
SELECT title, policy_type, version FROM privacy_policies ORDER BY policy_type;
```

## CMS User Guide

### Managing Hero Slides

1. Navigate to **CMS → Settings → Hero Slides**
2. View current slides with image preview and "X / 4" counter
3. Click **"Add Slide"** to create a new slide (max 4)
4. Fill in title, subtitle, image URL, CTA text/link, display order
5. Click **"Save"** to create
6. Edit or delete slides as needed
7. Toggle active/inactive to enable/disable slides
8. Changes reflect immediately on the homepage

### Managing Special Deals

1. Navigate to **CMS → Settings → Deals**
2. View current deals with badge color swatches and "X / 3" counter
3. Click **"Add Deal"** to create (max 3)
4. Fill in: title, subtitle, description, image URL, badge text/color, link URL, countdown label
5. Save, edit, delete, and toggle as needed

### Managing Testimonials

1. Navigate to **CMS → Settings → Testimonials**
2. Click **"Add Testimonial"** to create
3. Fill in: name, role, company, avatar URL, rating (1-5), quote, project
4. Save to update the homepage testimonial rotator

### Managing Privacy Policies

1. Navigate to **CMS → Settings → Policies**
2. View privacy, terms, and cookie policies
3. Click **"Add Policy"** or edit existing
4. Set: title, content (markdown), policy type, version, effective date
5. Changes reflect on `/privacy`, `/terms`, and `/cookie-policy` pages

### Product Quantity Configuration

1. Navigate to **CMS → Products → [Product] → Edit**
2. Find the **"Quantity Configuration"** section in the sidebar
3. Toggle **"Allow Manual Quantity Entry"** to enable text input
4. If enabled, set **"Quantity Interval"** (e.g., 5 for steps of 5)
5. If disabled, set **"Quantity Presets"** (one value per line, e.g., "50\n100\n250")
6. Save to update the product detail page quantity selector

### Product Image Reordering

1. Navigate to **CMS → Products → [Product]**
2. In the **Images** section, drag images by the grip handle to reorder
3. Order saves automatically
4. Click the star icon to set an image as primary thumbnail

## Auth Configuration Reference

### OAuth Setup

The project uses `better-auth` with social providers. To enable OAuth:

1. Create OAuth apps at:
   - [Google Cloud Console](https://console.cloud.google.com) — OAuth 2.0 credentials
   - [Facebook Developers](https://developers.facebook.com) — Facebook Login
   - [TikTok Developers](https://developers.tiktok.com) — TikTok Login

2. Set environment variables:
   ```
   GOOGLE_CLIENT_ID=xxx
   GOOGLE_CLIENT_SECRET=xxx
   FACEBOOK_CLIENT_ID=xxx
   FACEBOOK_CLIENT_SECRET=xxx
   TIKTOK_CLIENT_ID=xxx
   TIKTOK_CLIENT_SECRET=xxx
   ```

3. OAuth callback URLs follow `better-auth` convention:
   ```
   http://localhost:3000/api/auth/callback/google
   http://localhost:3000/api/auth/callback/facebook
   http://localhost:3000/api/auth/callback/tiktok
   ```

### Email Verification

- Enabled via `better-auth`'s `emailVerification` configuration
- Verification email sent on signup (`sendOnSignUp: true`)
- Auto sign-in after verification enabled
- Custom email template in `lib/email-templates/email-verification.ts`

### Password Reset

- Real SMTP-based password reset via `better-auth`'s `sendResetPassword` callback
- Uses `nodemailer` with existing SMTP configuration
- Custom email template in `lib/email-templates/password-reset.ts`

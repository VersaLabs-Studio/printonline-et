## Executive Summary

Version 3.6 completes the **data management layer** that was deferred from v2. Every data point that was previously hardcoded, placeholder-only, or missing a CMS interface is now fully CRUD-capable inside the admin panel. This is a **backend + CMS-focused release** — there are no storefront UI changes.

**Key Highlights:**
- **Products CRUD** — Full create/edit/delete with all fields (overview, rush eligibility, quantity thresholds)
- **Categories CRUD** — Full create/edit/delete with reorder capability
- **Product Options & Values** — Dedicated management page for configurable product options
- **Pricing Matrix** — Row-level CRUD with CSV bulk import
- **Designer Fee Tiers** — Quantity-based tiered pricing for designer services
- **Site Settings** — Global configuration table (rush fee, delivery thresholds, etc.)
- **Delivery Zones** — Database-backed, CMS-configurable zone management
- **Customer Account Settings** — Self-service profile and password management

**What Changed Since v3.5:**

v3.5 focused on customer-facing features (messaging, delivery pricing, account deletion). v3.6 shifts focus to the **admin/CMS side**, enabling the PrintOnline team to manage all product, pricing, and configuration data through the CMS interface instead of requiring direct database access.

| Module | Before v3.6 | After v3.6 |
|--------|-------------|------------|
| Products | "Under Maintenance" page | Full CRUD with all fields |
| Categories | "Under Maintenance" page | Full CRUD with reorder |
| Product Options | Visual-only display | Full CRUD management |
| Pricing Matrix | Table exists, no UI | Full CRUD + bulk import |
| Designer Fees | Flat fee only | Quantity-tiered pricing |
| Delivery Zones | Hardcoded in code | DB-backed, CMS-configurable |
| Site Settings | None | Global config table |
| Account Settings | Read-only | Self-service CRUD |

## Migration Guide

### Prerequisites

- Access to Supabase Dashboard (SQL Editor)
- Service role or postgres role permissions
- All prior migrations (001-018) already applied

### Migration Files

Run these SQL scripts in the Supabase SQL Editor **in this exact order**:

| # | Migration File | What It Does | Safe to Re-run? |
|---|----------------|--------------|-----------------|
| 1 | `019_site_settings.sql` | Creates `site_settings` table + indexes + RLS + trigger | No (will fail if exists) |
| 2 | `020_delivery_zones.sql` | Creates `delivery_zones` + `delivery_quantity_tiers` tables | No (will fail if exists) |
| 3 | `021_designer_fee_tiers.sql` | Creates `designer_fee_tiers` table | No (will fail if exists) |
| 4 | `022_product_schema_additions.sql` | Adds columns to `products` table | Yes (uses IF NOT EXISTS) |
| 5 | `023_rls_admin_policies.sql` | Adds admin RLS policies | Yes (uses IF NOT EXISTS) |

### Seed Data

After all 5 migrations, run the seed file:

| # | Seed File | What It Does |
|---|-----------|--------------|
| 6 | `014_site_settings_and_delivery_zones.sql` | Populates default settings and 10 delivery zones |

### Verification Queries

After running all migrations + seed, execute these queries to verify:

```sql
-- 1. Verify site_settings (expect 5 rows)
SELECT setting_key, setting_value, label, category FROM site_settings ORDER BY category, setting_key;

-- 2. Verify delivery_zones (expect 10 rows)
SELECT sub_city, base_fee, zone_label, is_active FROM delivery_zones ORDER BY display_order;

-- 3. Verify delivery_quantity_tiers table (expect 0 rows)
SELECT COUNT(*) FROM delivery_quantity_tiers;

-- 4. Verify designer_fee_tiers table (expect 0 rows)
SELECT COUNT(*) FROM designer_fee_tiers;

-- 5. Verify products has new columns (expect 3 rows)
SELECT column_name, data_type, column_default 
FROM information_schema.columns 
WHERE table_name = 'products' 
AND column_name IN ('overview', 'rush_eligible', 'quantity_thresholds');

-- 6. Verify RLS policies exist
SELECT tablename, policyname FROM pg_policies 
WHERE tablename IN ('site_settings', 'delivery_zones', 'delivery_quantity_tiers', 'designer_fee_tiers');

-- 7. Verify triggers exist
SELECT trigger_name, event_object_table FROM information_schema.triggers 
WHERE event_object_table IN ('site_settings', 'delivery_zones');
```

### Troubleshooting

| Issue | Fix |
|-------|-----|
| `relation "site_settings" already exists` | Migration already ran — skip to next |
| `function update_updated_at() does not exist` | Run `008_functions.sql` first |
| `permission denied` | Use SQL Editor with service_role or postgres role |
| Seed `ON CONFLICT` errors | Data already exists — safe to ignore |

## CMS User Guide

### Managing Products

1. Navigate to **CMS → Products**
2. Browse the product list with search and filters
3. Click **"New Product"** to create a product
4. Fill in all required fields (name, slug, category, price)
5. Optionally fill in overview, rush eligibility, and quantity thresholds
6. Click **"Save"** to create the product
7. Click on any product to view its detail page
8. Use the **Options**, **Pricing**, and **Fees** buttons to manage sub-features

### Managing Categories

1. Navigate to **CMS → Categories**
2. Click **"New Category"** to create a category
3. Fill in name (slug auto-generates), description, and image URL
4. Click **"Save"** to create
5. Click the edit icon on any category to modify it
6. Click the delete icon to remove a category (with confirmation)

### Managing Product Options

1. Navigate to **CMS → Products → [Product] → Options**
2. Click **"Add Option Group"** to create a new option
3. Fill in label, key, and field type (select, radio, checkbox, etc.)
4. Click **"Add Value"** on an option to add values
5. Fill in label, value, and optional price adjustment
6. Edit or delete options and values as needed

### Managing Pricing Matrix

1. Navigate to **CMS → Products → [Product] → Pricing**
2. Click **"Add Entry"** to add a pricing row
3. Fill in matrix key, label, and price
4. For bulk import, click **"Bulk Import"** and paste CSV data
5. Use **"Clear All"** to remove all entries (with confirmation)

### Managing Designer Fee Tiers

1. Navigate to **CMS → Products → [Product] → Designer Fees**
2. Click **"Add Tier"** to create a fee tier
3. Fill in min quantity, max quantity, fee amount, and optional label
4. Edit or delete tiers as needed

### Managing Site Settings

1. Navigate to **CMS → Settings**
2. View settings grouped by category (Pricing, Delivery, General)
3. Click the edit icon next to any setting
4. Modify the value and click **"Save"**
5. Changes take effect immediately

### Managing Delivery Zones

1. Navigate to **CMS → Settings → Delivery Zones**
2. View all zones in the table
3. Click **"Add Zone"** to create a new zone
4. Fill in sub-city name, base fee, and optional label
5. Edit or delete zones as needed
6. Toggle active/inactive to enable/disable zones

### Customer Account Settings

1. Log in as a customer
2. Navigate to **Account → Settings**
3. Update profile information (name, phone, company, address)
4. Click **"Save Profile"** to persist changes
5. Use the **"Change Password"** section to update your password

## Technical Details

### Architecture

v3.6 follows the established architecture patterns:

- **API Routes** — All CMS routes use `auth.api.getSession()` + `isAdmin()` + Zod validation + `supabaseAdmin`
- **Hooks** — TanStack Query hooks for data fetching, mutations with cache invalidation
- **Components** — shadcn/ui components with consistent styling
- **Types** — Supabase auto-generated types + manually appended convenience aliases

### File Manifest

**New Files (44+):**
- 5 migration files (`019-023`)
- 1 seed file
- 16 API routes
- 9 CMS pages
- 12 CMS components
- 5 hook files
- 1 UI component (switch)

**Modified Files (14):**
- Types, validations, delivery calculator, existing CMS pages/hook updates

### Security

- All CMS API routes require admin authentication
- Account routes require user authentication
- RLS policies enforced on all new tables
- Input validation via Zod schemas on all write endpoints
- No secrets or keys exposed in client-side code
